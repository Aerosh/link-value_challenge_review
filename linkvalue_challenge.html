Review Challenge¶

Définition du problème¶

Dans ce challenge, nous allons explorer divers mthodes afin de prdire si la review d'un utilisateur peut tre utile pour les autres utilisateurs.

Ici, il s'agit d'une prdiction binaire uniquement, "utile" ou "inutile", et nous disposons de review de PriceMinister.

Dans un premier temps, nous allons explorer le dataset fourni afin de dterminer les lments cls ncessaire afin d'obtenir un modle adquat notre problmatique. Nous allons notamment slectionner et construire les diffrentes features dont nous aurions besoin.

Ensuite, nous allons passer la partie de classification, avec la slection et la comparaison de plusieurs modles et mthodes, afin d'avoir une vue d'ensemble sur les choix effectuer.

Enfin, nous finirons sur une discussion des diffrents rsultats et une mise en place des diffrents lments pour la submission.

def train_eval_class(classifier, train_x, train_y, val_x, val_y_true):
    #training classifier with training data
    clf = classifier.fit(train_x, train_y)

class ItemSelector(BaseEstimator, TransformerMixin):
    """For data grouped by feature, select subset of data at a provided key.
    Extract column from panda DataFrame

    Parameters
    ----------
    key : hashable, required
        The key corresponding to the desired value in a mappable.
    """
    def __init__(self, key):
        self.key = key

    def fit(self, x, y=None):
        return self

    def transform(self, data_dict):
        return data_dict[self.key]
    
class TextStats(BaseEstimator, TransformerMixin):
    """ For text type data, retrieve number of characters, words and sentences, usually
        used in radibilty mesures.
    """

    def fit(self, x, y=None):
        return self

    def transform(self, texts):
        """
        Return number of words, characters and sentences, but also text length.
        Return in form of list of dict.
        """
        return [
            {
                "characters" : len(text) - text.count(' '),
                "words": len(re.findall(r"[\w']+", text)),
                "sentences": text.count('.') + text.count('!') + text.count('?'),
                "length" : len(text)
            }
            for text in texts
        ]

TF-IDF¶

Wordcount nous a permis d'avoir le nombre d'occurance des mots au sein d'un texte. Cependant, la longueur du texte aura une grande influence sur le nombre de mots qui apparaissent ou mme les rptitions. Or, nous avons vu que la longueur n'avait pas une grande influence sur l'utilit des review. Un autre problme est celui du vocabulaire commun tous les textes, aussi bien utile que inutile. En effet, il semblerait logique de mettre en avant les termes plutt "uniques" des bonnes reviews que ceux communs tous.

Pour cela, nous utilisons la technique TF-IDF pour "Term Frequency and Inverse Document Frequency" qui permet non seulement de prendre en compte l'occurence de certains terme en fonction de la longeur du texte pour en retir la frquence de ce term dans le document, mais aussi de pnaliser les termes communs plusieurs textes.

Latent Sementic Analysis¶

LSA est une mthode de rduction de feature qui permet de relier les diffrents termes et documents des "conceptes", assez similair au variables latentes dans les techniques de recommendation bas sur la factorisation de matrice.

LSA applique une Singular Value Decomposition (SVD) la matrice TF-IDF. Cela plusieurs avantages, notamment celui de rduite le nombre de dimension un subset qui est beaucoup plus "descriptif". Nous avons donc un matrix features plus dense.

Certaines documentations affirment que la SVD rduirait le bruit de notre matrices et la densit apporterait une amlioration de la classification. Un avantage personnel que je constate, c'est la transformation de donne dite "dures" (fquence de mots, ...) des donnes plus "abstraites (conceptes, sujets, ...) qui dcoulerait plus d'une analyse de texte que d'une simple lecture froide sans rflexion.

Conclusion¶

Deuximement, la plupart des algorithmes de classification utiliss ont commun point commun d'tre linaire, assumant donc que nos donnes sont linairement sparable.
